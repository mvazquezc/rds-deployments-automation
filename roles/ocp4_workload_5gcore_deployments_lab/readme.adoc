= ocp4_workload_5gcore_deployments_lab - Deploys 5G Core Deployments Lab

== Role overview

* This role automates the preparation of a **5G Core** lab environment on OpenShift. It consists of the following task files:

** Tasks: link:./tasks/pre_workload.yml[pre_workload.yml] - Sets up the hypervisor environment for the 5G Core lab deployment.
*** Validates system resources (memory, CPU, OS).
*** Configures storage, networking, dnsmasq, container registry, Gitea, web cache, and MinIO.
*** Mirrors OCP releases 4.18/4.19/4.20 and Core RDS operator indexes to the local registry via `oc-mirror`.
*** Creates 12 MNO VMs (3 masters + 9 workers in 3 failure domains) using kcli.
*** Debug task will print out: `pre_workload Tasks completed successfully.`

** Tasks: link:./tasks/workload.yml[workload.yml] - Deploys the Hub cluster and ArgoCD.
*** Deploys the OpenShift Hub cluster using the local disconnected registry (pre-populated by oc-mirror).
*** Configures ArgoCD, Multi-Cluster Hub (MCH), and Multi-Cluster Engine (MCE).
*** Does **not** deploy the MNO cluster; the user deploys it via ClusterInstance and PolicyGenerators from the lab repo.
*** Debug task will print out: `workload Tasks completed successfully.`

** Tasks: link:./tasks/post_workload.yml[post_workload.yml] - Saves user credentials and lab access information.
*** Debug task will print out: `post_workload Tasks completed successfully.`

** Tasks: link:./tasks/remove_workload.yml[remove_workload.yml] - Removes the lab environment.
*** Deletes the kcli Hub plan (Hub VM + 12 MNO VMs) via `kcli delete plan hub --yes`.
*** Stops MinIO if `lab_deploy_minio` was true.
*** Debug task will print out: `remove_workload Tasks completed successfully.`

== Review the defaults variable file

* This file link:./defaults/main.yml[./defaults/main.yml] contains all the variables you need to define to control the deployment of your workload.
* The variable *ocp_username* is mandatory to assign the workload to the correct OpenShift user.
* A variable *silent=True* can be passed to suppress debug messages.
* Key variables:
** *lab_version*: Branch/tag of the 5g-core-deployments-on-ocp-lab repository (default: `main`).
** *repo_user*: GitHub user for the lab repository (default: `alosadagrande`).
** *lab_release*: Target OCP release (default: `4.20`).
** *lab_hub_vm_cpus*: CPU cores for the Hub VM (default: `16`).
** *lab_hub_vm_memory*: Memory for the Hub VM in MB (default: `48000`).
** *lab_hub_vm_disk*: Disk for the Hub VM in GB (default: `200`).
** *lab_mno_master_count*: Number of MNO master VMs (default: `3`).
** *lab_mno_worker_count*: Number of MNO worker VMs (default: `9`).
** *lab_mno_failure_domains*: Number of failure domains, 3 workers each (default: `3`).
** *lab_mno_master_vm_cpus* / *lab_mno_master_vm_memory* / *lab_mno_master_vm_disk*: Resources per master VM.
** *lab_mno_worker_vm_cpus* / *lab_mno_worker_vm_memory* / *lab_mno_worker_vm_disk*: Resources per worker VM.
** *lab_mno_worker_vm_extra_disk*: Extra disk per worker VM for ODF (default: `120` GB).
** *lab_deploy_minio*: Deploy MinIO for RHACM multi-cluster observability S3 storage (default: `true`).
** *run_oc_mirror*: Mirror OCP releases and operators to the local registry via oc-mirror (default: `true`; set `false` if the registry is pre-populated).
** *oc_mirror_timeout*: Async timeout in seconds for the oc-mirror run (default: `7200`).
** *imageset_config_file*: ImageSetConfiguration file used by oc-mirror (default: `imageset-mirror-core.yaml`).
** *disconnected_update*: Keep `false`; oc-mirror handles mirroring (kcli does not mirror releases).

* You can modify any of these default values by adding `-e "variable_name=variable_value"` to the command line.

=== Deploy a Workload with the `ocp-workload` playbook [Mostly for testing]

----
TARGET_HOST="your-hypervisor.example.com"
OCP_USERNAME="your-user"
WORKLOAD="ocp4_workload_5gcore_deployments_lab"
GUID=1001

# TARGET_HOST is specified in the command line, without using an inventory file
ansible-playbook -i ${TARGET_HOST}, ./configs/ocp-workloads/ocp-workload.yml \
    -e "ansible_ssh_private_key_file=~/.ssh/keytoyourhost.pem" \
    -e "ansible_user=root" \
    -e "ocp_username=${OCP_USERNAME}" \
    -e "ocp_workload=${WORKLOAD}" \
    -e "silent=False" \
    -e "guid=${GUID}" \
    -e "ACTION=create" \
    -e "ocp4_pull_secret=$(cat pull-secret.json)"
----
